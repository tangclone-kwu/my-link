# Product Requirements Document (PRD)

## 1. 프로젝트 개요

### 1.1. 프로젝트 명
**마이링크 (My Link)**

### 1.2. 목적
사용자가 자신을 나타내는 다양한 소셜 미디어, 포트폴리오, 블로그, 외부 서비스 링크들을 단일 URL(프로필 페이지)로 모아 쉽고 깔끔하게 공유할 수 있는 다중 링크 큐레이션 서비스를 제공합니다.

### 1.3. 대상 사용자 (Target Audience)
- **크리에이터/아티스트:** 자신의 작품, 후원 채널, 소셜 미디어 플랫폼을 팬들과 한곳에서 공유하고자 하는 사람.
- **프리랜서 및 구직자:** 포트폴리오, 이력서, 링크드인, 개인 블로그 등 자신의 전문성을 나타내는 링크를 한데 묶어 제공하고자 하는 사람.
- **소상공인 및 소규모 브랜드:** 인스타그램 등 단일 링크만을 허용하는 프로필에 웹사이트, 카카오톡 채널, 스마트스토어 등을 한 번에 연결하고자 하는 시람.

---

## 2. 제품 정책 및 주요 특징

- **사용자 인증 (Auth):** 복잡한 회원가입 절차 없이, **Firebase Authentication을 통한 Google 소셜 로그인**만 지원합니다.
- **수익화 모델 (Monetization):** 모든 기능을 과금 없이 누릴 수 있는 **완전 무료 서비스**로 운영됩니다.
- **프로필 URL 구조:** 배포된 애플리케이션의 커스텀 도메인(예: `[도메인]/닉네임`) 형태로 오직 닉네임 기반의 유니크한 프로필 주소를 제공합니다 (영문/숫자 허용, 별도 handle 사용 안함).

---

## 3. 핵심 기능 목록 (Feature List)

### 3.1. 필수 기능 (Must-Have)
서비스가 정상적으로 작동하기 위해 반드시 포함되어야 하는 핵심 기능입니다.

1. **회원가입 및 로그인 (Authentication)**
   - Firebase 기반 구글 소셜 간편 로그인.
2. **랜딩 페이지 (프로필 뷰어)**
   - `[커스텀도메인]/username` 형태로 제공되는 최종 프로필 페이지.
3. **링크 관리 버추얼 대시보드 (Link Management)**
   - 링크 추가, 수정, 삭제 기능. (순서 변경, 드래그 드롭 기능 없음)
   - 링크 항목(Link item) 썸네일로 **구글 파비콘 API(Google Favicon API)**를 이용해 파비콘을 자동 추출 및 적용.
   - 링크 텍스트 및 URL 수정 시 모달창이 아닌 **인라인 편집(Inline Editing)** 방식 적용.
4. **프로필 설정 (Profile Customization)**
   - 사용자 닉네임(`DisplayName`) 및 짧은 자기소개(Bio) 작성을 **인라인 편집**으로 제공.
   - 닉네임은 언제든 수정 가능.
   - (프로필 이미지 업로드 기능 없음, 테마/꾸미기 기능 없음)

### 3.2. 부가 기능 (Future Scope)
*(방문자 통계 등 부가 서비스는 향후 추가 업데이트 기능으로 논의함)*

1. **개별 링크 클릭 수 조회 (Analytics)**
   - 추후 업데이트를 통해 **개별 링크별 클릭 조회수**만 간단하게 제공.

## 4. 기능 상세 설명 및 DB 모델링 (Feature & Data Details)

### 4.1. 데이터베이스 모델링 (Database Modeling)
Firebase Firestore를 기준으로, 사용자 프로필 정보와 링크 데이터를 분리하여 **서브 컬렉션(Sub-collections)** 구조로 관리합니다.

- **`users` (Collection)**
  - `uid` (Document ID)
  - `nickname` (프로필 URL에 들어갈 고유 문자열 및 화면에 노출될 이름, 언제든 수정 가능)
  - `bio` (소개글)
  - **`links` (Sub-collection)**
    - `linkId` (Document ID)
    - `title` (버튼 텍스트)
    - `url` (이동할 주소)
    - `createdAt` (생성일자)
    - *(추후 업데이트용)* `clickCount` (클릭 조회수, 기본값 0)

### 4.2. 링크 관리 및 프로필 설정 
- **데이터 흐름:** 
  - 서브 컬렉션 `users/{uid}/links` 및 `users/{uid}` 문서에서 데이터를 인라인 수정 방식으로 직접 업데이트합니다.
- **인터랙션 및 시각화:**
  - **인라인 편집:** 닉네임, 소개글, 링크 제목, 주소 등을 클릭 시 즉시 입력창(Input)으로 전환되어 바로 수정할 수 있도록 하여 편집 동선을 최소화합니다.
  - **자동 파비콘 매칭 (Google API):** 사용자가 등록한 URL을 `http://www.google.com/s2/favicons?domain=[도메인]` 형태로 호출하여 별도 저장 없이 파비콘 썸네일을 노출합니다.
  - 별도의 디자인 테마 커스터마이징은 배제하고, 통일된 기본 UI(글로벌 스타일) 하나만 제공합니다.
