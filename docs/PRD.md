# Product Requirements Document (PRD)

## 1. 프로젝트 개요

### 1.1. 프로젝트 명
**마이링크 (My Link)**

### 1.2. 목적
사용자가 자신을 나타내는 다양한 소셜 미디어, 포트폴리오, 블로그, 외부 서비스 링크들을 단일 URL(프로필 페이지)로 모아 쉽고 깔끔하게 공유할 수 있는 다중 링크 큐레이션 서비스를 제공합니다.

### 1.3. 대상 사용자 (Target Audience)
- **크리에이터/아티스트:** 자신의 작품, 후원 채널, 소셜 미디어 플랫폼을 팬들과 한곳에서 공유하고자 하는 사람.
- **프리랜서 및 구직자:** 포트폴리오, 이력서, 링크드인, 개인 블로그 등 자신의 전문성을 나타내는 링크를 한데 묶어 제공하고자 하는 사람.
- **소상공인 및 소규모 브랜드:** 인스타그램 등 단일 링크만을 허용하는 프로필에 웹사이트, 카카오톡 채널, 스마트스토어 등을 한 번에 연결하고자 하는 사람.

---

## 2. 필수 질문 및 확인 사항 (User Clarification Needed)
프로젝트 구체화를 위해 기획 단계에서 확정해야 할 몇 가지 질문입니다.

> [!IMPORTANT]
> **Q1. 사용자 인증(Auth) 방식:** 이메일/비밀번호 기반의 자체 회원가입을 지원할 것인지, 혹은 Google/Kakao 등의 소셜 로그인(OAuth)만 지원할 것인지 결정이 필요합니다.
> **Q2. 요금제 모델 (Monetization):** 완전 무료 서비스인지, 혹은 링크 개수 무제한 / 커스텀 테마 / 프리미엄 분석(Analytics) 기능 등을 묶어 `Free`와 `Pro` 티어로 구분할 예정인지 궁금합니다.
> **Q3. 프로필 URL 구조:** 유저의 링크 페이지 주소를 `mylink.com/아이디` 형태로 제공할 예정인지, 아니면 다른 도메인 전략이 있는지 확인 부탁드립니다.

## 3. 핵심 기능 목록 (Feature List)

### 3.1. 필수 기능 (Must-Have)
서비스가 정상적으로 작동하기 위해 반드시 포함되어야 하는 핵심 기능입니다.

1. **회원가입 및 로그인 (Authentication)**
2. **랜딩 페이지 (프로필 뷰어)**
   - 방문자가 유저의 고유 URL 주소를 통해 접속했을 때 보이는 최종 페이지.
3. **링크 관리 버추얼 대시보드 (Link Management)**
   - 링크 추가, 수정, 삭제 기능.
   - 드래그 앤 드롭으로 링크 노출 순서 변경 기능.
   - 링크별 활성화/비활성화 토글 기능.
4. **프로필 설정 (Profile Customization)**
   - 프로필 사진 업로드, 사용자 이름(혹은 닉네임), 짧은 자기소개(Bio) 작성 기능.

### 3.2. 선택 및 부가 기능 (Should-Have / Could-Have)
서비스의 완성도와 사용자 경험을 높여주지만 초기 런칭 필수 요건은 아닌 기능입니다.

1. **테마 커스터마이징 (Theme Customization)** (Should-Have)
   - 템플릿 색상, 버튼 스타일(예: 네오브루탈리즘, 라운드, 스퀘어 등), 폰트 변경 기능.
2. **트래픽 통계 및 분석 (Analytics)** (Should-Have)
   - 방문자 수(PV, UV) 및 개별 링크 클릭 수 통계 제공.
3. **QR 코드 생성 (QR Code Generation)** (Could-Have)
   - 프로필 페이지로 직결되는 오프라인 공유용 QR 코드 자동 생성.
4. **예약 및 만료 링크 설정 (Scheduled Links)** (Could-Have)
   - 특정 날짜와 시간에만 노출되거나 사라지도록 설정하는 기능.

## 4. 기능 상세 설명 (Feature Details)

### 4.1. 링크 관리 버추얼 대시보드
- **데이터 구조:** 각 링크 엔티티(Entity)는 `ID`, `Title`(사용자에게 보이는 버튼 이름), `URL`(이동할 실제 주소), `Order`(정렬 순서), `isActive`(노출 여부) 속성을 갖습니다.
- **인터랙션:** Dashboard 페이지에서 실시간 미리보기(Live Preview) 컴포넌트를 제공하여, 링크를 수정하거나 순서를 바꿀 때 우측 화면(또는 모달창)에서 프로필 페이지의 변화를 즉각적으로 보여줍니다.

### 4.2. 프로필 설정 및 테마 커스터마이징
- 프로필 레이아웃은 기본적으로 상단에 `[프로필 이미지] -> [이름] -> [소개글]` 순서로 배치됩니다.
- 테마 에디터에서 CSS Variable이나 Tailwind 설정 값(Colors, Border Radius 등)을 조작하여 유저의 고유한 브랜드 아이덴티티를 살릴 수 있도록 설계합니다.

### 4.3. 통계 및 분석 (개선 제안)
단순한 클릭 수 수집을 넘어서 **'어떤 소셜 플랫폼을 통해 트래픽이 유입되었는지' (Referrer)** 정보를 간단한 차트로 보여주면 사용자의 마케팅 효과 분석에 큰 도움이 됩니다.

## 5. 추가 제안 (Improvement Suggestions)

1. **마이크로 애니메이션 강화:** 현재 적용된 '네오브루탈리즘' 디자인 철학을 이어받아, 관리자 대시보드에서도 버튼 클릭이나 드래그 앤 드롭 시 통통 튀는 물리적 피드백(Spring animation)을 넣으면 사용성이 훨씬 좋아집니다.
2. **미리보기 반응형 토글:** 대시보드 내 실시간 프로필 미리보기 화면 위에 `Mobile` / `Desktop` 토글 버튼을 두어, 사용자가 여러 디바이스에서 어떻게 보이는지 쉽게 확인할 수 있게 하는 것을 추천합니다.
3. **소셜 아이콘 세트 내장:** 사용자가 직접 이메일, 인스타그램, 깃허브 URL을 입력하면 시스템이 자동으로 URL 패턴을 인식해 해당 서비스의 아이콘을 링크 버튼 옆에 렌더링해주면 훨씬 직관적인 UI가 됩니다.
