# 와이어프레임 (Wireframes)

"마이링크(My Link)" 서비스의 핵심 화면인 **[1. 대시보드(소유자 뷰)]**와 **[2. 프로필 뷰어(방문자 뷰)]**의 화면 레이아웃을 정의합니다.

---

## 1. 링크 관리 대시보드 (Dashboard - Owner)

소유자가 로그인하면 보이는 메인 화면입니다. 좌측은 설정 및 링크 편집 영역, 우측은 실시간 프로필 미리보기 화면으로 구성됩니다.

### 1-1. 화면 구성 (Desktop)

```text
+---------------------------------------------------------+
| [마이링크 로고]                         [로그아웃] 버튼 |
+---------------------------------------------------------+
| [좌측 패널: 링크 관리 및 설정] | [우측 패널: 미리보기]    |
|                                |                          |
| 1. 내 프로필 주소 노출 구역    | +----------------------+ |
|    mylink.com/[nickname] [복사]| |                      | |
|                                | | [닉네임]             | |
| 2. 내 프로필 정보 (인라인수정) | |                      | |
|    - 닉네임: [ dev123 ]        | | [소개글]             | |
|    - 소개글: [ 프론트엔드..]   | |                      | |
|                                | | +------------------+ | |
| 3. 링크 관리 영역              | | | [I] 내 블로그    | | |
|    [ + 새 링크 추가 ] 버튼     | | +------------------+ | |
|                                | | +------------------+ | |
|    +-------------------------+ | | | [I] 깃허브       | | |
|    | Title: [내 블로그]      | | | +------------------+ | |
|    | URL: [blog.com...] [🗑️]| | |                      | |
|    +-------------------------+ | |                      | |
|    +-------------------------+ | +----------------------+ |
|    | Title: [깃허브]         | |                          |
|    | URL: [github.com][🗑️]| |                          |
|    +-------------------------+ |                          |
|                                |                          |
+---------------------------------------------------------+
```

### 1-2. 기능 및 컴포넌트 상세
- **프로필 주소 및 복사:** `[도메인]/nickname`이 항상 상단에 표시되며 우측 네모 아이콘(복사) 클릭 시 토스트 메시지가 뜹니다.
- **프로필 및 링크 인라인 수정:** 닉네임, 소개글, 링크 Title, URL 항목은 텍스트처럼 보이지만, 클릭 시 `Input` 필드로 전환되어 그 자리에서 즉시 텍스트를 수정할 수 있습니다.
- **삭제 버튼 (`[🗑️]`):** 링크 리스트 우측 끝에 위치하며, 클릭 시 모달창 없이 1초만에 화면과 DB에서 즉시 삭제됩니다.
- **우측 미리보기:** 좌측 패널에서 수정이 발생할 때마다, 우측 가상 모바일 프레임(미리보기) 화면이 실시간 반영되어 디자인이 어떻게 적용되는지 보여줍니다.

---

## 2. 프로필 뷰어 (Profile Page - Visitor)

방문자가 `mylink.com/dev123` 등 소유자의 고유 URL로 접속했을 때 보이는 최종 결과 화면입니다. 네오브루탈리즘 테마의 단일 글로벌 디자인이 적용됩니다.

### 2-1. 화면 구성 (Mobile First)

```text
+-------------------------------+
|                               |
|          [ dev123 ]           | <- 닉네임 (가장 큰 폰트)
|                               |
|   "프론트엔드 개발자입니다."  | <- 소개글(Bio)
|                               |
|                               |
|  +-------------------------+  |
|  | [B] 내 블로그           |  | <- 버튼을 두꺼운 검은 테두리와
|  +-------------------------+  |    뚜렷한 그림자로 강조
|                               |
|  +-------------------------+  |
|  | [G] 깃허브              |  | <- [G], [B] 자리는 
|  +-------------------------+  |    구글 Favicon API가 불러온
|                               |    자동 아이콘(썸네일)
|  +-------------------------+  |
|  | [P] 이력서 및 포트폴리오|  |
|  +-------------------------+  |
|                               |
|     POWERED BY MY LINK.       |
|                               |
+-------------------------------+
```

### 2-2. 기능 및 컴포넌트 상세
- **테마:** 모든 유저는 개별 커스텀 없이, 서비스가 제공하는 굵은 선(Thick border), 비비드 컬러, 뚜렷한 그림자를 지닌 네오브루탈리즘(Neobrutalism) 단일 디자인으로 고정됩니다.
- **파비콘 렌더링:** 소유자가 등록한 URL의 루트 도메인을 분석해 자동으로 파비콘 아이콘 영역(`[B]`, `[G]`)에 삽입하여 가독성을 높입니다.
- **호버 애니메이션:** 방문자가 링크 버튼 위로 마우스를 올릴 시, 오프셋 그림자가 줄어들고 버튼이 아래로 살짝 밀리는 물리적 누름 애니메이션을 적용하여 사용 감각을 높입니다.
